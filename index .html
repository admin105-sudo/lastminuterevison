Linux Boot Process
5 stages • High priority
▾
QLinux eppadi boot aaguthu?
BIOS/UEFI → GRUB → Kernel → systemd → Login

BIOS/UEFI: Hardware check — POST test
GRUB: Kernel-ai memory-la load pannuvadhu
Kernel: Drivers init, root filesystem mount
systemd: PID 1 — parallel services start
Login: User login ready
💡 One-liner: "BIOS → GRUB → Kernel → systemd → Login"
Qsystemd vs init difference?
Old init = Sequential start → SLOW
systemd = Parallel start → FAST boot, PID 1 
systemctl start nginx
systemctl enable nginx
systemctl status nginx
💡 systemd = PID 1, parallel, modern init
QRoot password reset eppadi?
GRUB menu → "e" → linux line-la rd.break add → Ctrl+X 
mount -o remount,rw /sysroot
chroot /sysroot
passwd root
touch /.autorelabel
exit && reboot

Dynamic CloudWatch (CPU > 70% → add instance)
Scheduled Time-based (9AM scale up)
💡 Horizontal = cloud best practice. No downtime!
